# Session Capture

*v1.3 — Simplified artifact persistence, fixed pruning, added skip-gate audit*

Captures key decisions, open questions, follow-ups, and working artifacts from the current session. Writes a structured entry to the session log and persists unsaved artifacts for cross-session continuity. Run at the end of any substantive work session.

## Arguments

`$ARGUMENTS` can include:
- *(none)* — full capture (decisions + questions + follow-ups + context)
- `quick` — abbreviated capture (decisions + follow-ups only, no context summary)
- `project` — also write a project-level `HANDOFF.md` to the project root
- `project:name` — tag the entry with a specific project name

## Instructions

### Step 1: Identify Session Scope

Review the conversation history and determine:
- **Session topic(s):** What was worked on? (1-2 line summary)
- **Project(s) touched:** Match against known projects
- **Duration indicator:** Brief (< 10 exchanges), Medium (10-30), Extended (30+)

### Step 2: Extract Session Content

Scan the conversation for:

**Decisions made** — things that were decided or confirmed:
- Configuration changes, skill edits, policy updates
- Approach selections (e.g., "we'll use method X instead of Y")
- Commitments (e.g., "will send email to X by Friday")

**Approaches rejected** *(optional — populate only if the session pivoted away from a started approach, or a tool/method was attempted, produced a result, and was discarded; skip for micro-adjustments):*
- [Approach] — [why rejected]

**Open questions** — unresolved items that need future attention:
- Questions raised but not answered
- Items deferred ("we'll handle that next time")
- Blockers or dependencies on external input

**Follow-ups** — concrete next steps:
- Tasks to do in a future session
- Things to check, test, or verify
- People to contact or documents to review

**Key artifacts created** — files written or modified:
- New skills, agents, or config files
- Documents drafted or edited
- State changes (e.g., emails sent, reminders modified)

### Step 2.5: Working Artifacts

**Skip if** ALL of: (a) <=5 exchanges, (b) no files created/modified, (c) no decisions logged, (d) no agents spawned. Also skip in `quick` mode. When skipped, append to session-log.md: `[YYYY-MM-DD HH:MM] -- SKIPPED artifact scan: [reason]`.

Scan for substantial working content not already saved to a file — restructuring plans, agent review outputs, draft content, detailed decision rationale. Do NOT save reproducible outputs (tool results, search output) or content already written to files.

For each artifact: save to `<project-root>/working-notes/` (if `project` arg and project root found) or `~/Documents/working-notes/`. Use `YYYY-MM-DD_description.md` naming with a brief header (title, source, usage hint). Preserve full content.

Budget: consolidate to 1-3 files max. Err toward saving — you can delete unwanted artifacts next session.

### Step 3: Write Session Entry

Append to `~/Documents/session-log.md` (or your preferred location):

```markdown
---

## [YYYY-MM-DD HH:MM] — [Session topic summary]
SESSLOG:[YYYY-MM-DD HH:MM]

**Project(s):** [project names or "General"]
**Duration:** [Brief/Medium/Extended]

### Decisions
- [Decision 1]
- [Decision 2]

[If approaches were rejected:]
### Approaches Rejected
- [Approach] — [why rejected]

### Open Questions
- [Question 1]
- [Question 2]

### Follow-ups
- [ ] [Action item 1]
- [ ] [Action item 2]

### Artifacts
- [Created/Modified] `path/to/file` — [what changed]

[If not `quick`:]
### Context
[2-4 sentence summary of key context a future session would need to pick up where this left off.]
```

If the file doesn't exist, create it with a header:
```markdown
# Session Log
*Auto-generated by /done. Reverse chronological — newest entries at top.*

```

**New entries go at the TOP** (after the header), reverse chronological.

### Step 4: Prune and Cleanup

**Session log pruning:**
1. Count lines in your session log file
2. If <=1,500 lines -> skip pruning
3. If >1,500 lines -> archive old entries:
   - Use lines matching `^## ` as entry boundaries
   - Move entries with dates older than 60 days to `session-log-archive.md` (append; create if needed with header `# Session Log Archive`)
   - **One-time migration:** if file exceeds 3,000 lines (backlog never pruned), move ALL entries older than 30 days
   - Report in summary: "Pruned [N] entries to archive."

**Working-notes cleanup:** Delete files in `working-notes/` older than 30 days (by mtime). Skip if no `working-notes/` directory exists.

### Step 5: Write Handoff Note

Write a condensed handoff file to `~/.claude/handoff.md` for next-session resumption. This file is read by a SessionStart hook so the next session has immediate context.

**Key rules:**
- Max **30 lines** total (Working Artifacts lines don't count — they're pointers)
- Working Artifacts section only appears when artifacts were saved
- Use **absolute paths** in Key Files; **checkboxes** for Next Steps
- In `quick` mode: omit the Context section
- Active Decisions: only decisions constraining the *next* session
- If nothing meaningful to hand off, skip and note in summary
- Include `SESSLOG:[YYYY-MM-DD HH:MM]` cross-reference to the session log entry
- **Project-level:** if `project` arg provided, also write `HANDOFF.md` to the project root with relative paths

```markdown
# Handoff — [YYYY-MM-DD]
SESSLOG:[YYYY-MM-DD HH:MM]

## Session Topic
[1-line summary]

## Active Decisions
- [Decision constraining next session]

## Key Files
- [absolute path to important file]

## Next Steps
- [ ] [Most important next step]
- [ ] [Other follow-ups]

## Working Artifacts
- [path/to/artifact] — [what it contains]

## Context
[1-2 sentences for next session to pick up seamlessly]
```

**Overwrite logging:** Before overwriting the handoff file, if the existing file is non-empty, append to session-log.md: `*Previous handoff overwritten: [topic from old handoff]*`.

Overwrite any existing `handoff.md` — only the most recent session's handoff matters.

### Step 6: Summary

Display a brief confirmation:

```
--------------------
SESSION CAPTURED
--------------------
Topic: [summary]
Decisions: [N] | Questions: [N] | Follow-ups: [N] | Artifacts: [N]
[If working artifacts saved:] Working artifacts saved: [N] files -> [directory path]
[If pruning ran:] Pruned: [N] entries -> session-log-archive.md
Logged to: session-log.md
Handoff: ~/.claude/handoff.md [written/skipped (trivial)]
[If project arg:] Handoff (project): [project-root]/HANDOFF.md [written]

[If open questions exist:]
!! [N] open questions -- pick these up next session

[If follow-ups exist:]
Next time: [most important follow-up item]
--------------------
```

## Customization Points

- **Log file location:** Change `~/Documents/session-log.md` to wherever you want session logs stored
- **Project matching:** Add logic to match against your known projects list (e.g., from CLAUDE.md or a goals file)
- **Handoff file location:** Change `~/.claude/handoff.md` if your SessionStart hook reads from a different path
- **Working-notes directory:** Change `~/Documents/working-notes/` to your preferred location
- **Archive retention:** Adjust the 60-day prune threshold or 1,500-line trigger as needed

## Error Handling

- **No substantive content:** If the session was just a quick lookup or greeting, report: "Nothing to capture — session was too brief." and exit.
- **Session log write fails:** Display the entry in the terminal so you can manually save it.

## Design Notes

- **Complements session-closing discovery** — that pattern suggests *improvements* (new skills, rules, agents). This skill captures *artifacts* (decisions, questions, follow-ups).
- **No MCP tools needed** — pure filesystem operation.
- **Skip-gate audit:** When working artifact scan is skipped, the reason is logged to session-log.md. Grep for `SKIPPED artifact scan` to detect over-firing.
- **SESSLOG cross-references:** Both session log entries and handoff notes include `SESSLOG:[YYYY-MM-DD HH:MM]` for grep-able traceability between the two.

Replace TOOL_CALLS with exact count and NOTES with brief summary (e.g., "3-decisions-2-followups-handoff").
